# 手表版小黑屋 — Claude Code 需求文档

> **本文档基于 A Dark Room 开源项目（MIT协议）源码完整分析而成。**
> **原版源码在 `adr/` 子文件夹中，可以参考但不要直接复制——原版依赖jQuery，我们要用纯原生JS重写。**

---

## 0. 项目概述

### 目标
将 A Dark Room 的核心玩法（生火间 + 森林）移植到Xiaomi Vela智能手表上。文件夹里面有Xiaomi Vela 应用程序模板

---

## 1. UI 布局

### 1.1 整体结构（单页纵向滚动）

```
┌──────────────────────────┐
│ [生火间] [森林]           │ ← 固定顶部标签栏（tab bar）
├──────────────────────────┤
│                          │
│  互动按钮区域             │ ← 当前场景的操作按钮
│  （最多同时显示4个）       │
│                          │
├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤ ← 分隔线
│                          │
│  数值面板                 │ ← 资源/状态数值
│  木头: 12   毛皮: 3      │
│                          │
├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤ ← 分隔线
│                          │
│  日志区域                 │ ← 事件文字，新的在上
│  "火堆熊熊燃烧."          │
│  "房间暖和."              │
│                          │
└──────────────────────────┘
```

### 1.2 标签栏（Tab Bar）
- 固定在视口顶部，`position: sticky; top: 0;`
- 初始只有"生火间"一个标签
- 当玩家有木头后解锁"森林"标签（原版逻辑：`typeof $SM.get('stores.wood') != 'undefined'`）
- 当前标签高亮（下划线或背景色区分）
- 触摸目标最小 44x44 逻辑像素

### 1.3 互动按钮区域
- 全宽块状按钮，纵向排列
- 按钮显示：操作名称 + 冷却倒计时（如果在冷却中）
- 冷却中的按钮变灰，显示剩余秒数
- 需要资源的按钮显示消耗（如 `建造陷阱 [木头:20]`）
- 资源不足时按钮禁用变灰
- **同一时间最多显示 4-5 个按钮**

### 1.4 数值面板
- 2列网格：`资源名: 数量`
- 只显示数量>0的资源
- 数值变化时短暂颜色闪烁：增加=绿 减少=红（CSS transition）

### 1.5 日志区域
- 新消息在最上方
- 新消息有 CSS opacity 淡入动画
- 最多保留 30 条，超出移除最旧的
- 重要事件可用不同颜色标记

### 1.6 配色方案

```css
:root {
  --bg: #000;
  --text: #b0b0b0;
  --text-bright: #e0e0e0;
  --text-dim: #555;
  --accent: #c8a000;    /* 火焰/重要操作 */
  --danger: #8b0000;    /* 危险 */
  --gain: #2e6e2e;      /* 获得 */
  --btn-bg: #1a1a1a;
  --btn-border: #333;
  --btn-disabled: #111;
  --divider: #222;
}
```

## 2. 游戏机制（完全还原原版）

### 2.1 火焰系统

**火焰状态**（原版 `Room.FireEnum`）：

| 值 | 英文 | 中文 |
|----|------|------|
| 0  | Dead | 火堆熄灭了. |
| 1  | Smoldering | 火堆冒着烟. |
| 2  | Flickering | 火堆摇曳不定. |
| 3  | Burning | 火堆熊熊燃烧. |
| 4  | Roaring | 火堆烈焰冲天. |

**机制**：
- 点击"添柴"（stoke fire）使火焰等级 +1（最高4）
- 添柴冷却时间：**10秒**（原版 `_STOKE_COOLDOWN: 10`）
- 添柴消耗：有木头时消耗木头，没木头时也能添（空手拨火）
- 火焰自然衰减：每 **5分钟** 降一级（原版 `_FIRE_COOL_DELAY: 5 * 60 * 1000`）
- **注意**：手表版可以考虑将衰减时间缩短到 2-3 分钟以适配碎片化体验

**房间温度**（原版 `Room.TempEnum`）：

| 值 | 英文 | 中文 |
|----|------|------|
| 0  | Freezing | 房间很冷. |
| 1  | Cold | 房间有点凉. |
| 2  | Mild | 房间暖和. |
| 3  | Warm | 房间很热. |
| 4  | Hot | 房间很宜人. |

- 房间温度每 **30秒** 趋近于火焰状态（原版 `_ROOM_WARM_DELAY: 30 * 1000`）
- 温度影响建造者NPC的行为

### 2.2 建造者（Builder）NPC

**触发条件**：
- 保持火焰 ≥ Burning（3）一段时间后触发
- 原版从火焰首次被点亮后约15秒开始检测（`_NEED_WOOD_DELAY: 15 * 1000`）

**建造者状态**（`_BUILDER_STATE_DELAY: 0.5 * 60 * 1000` = 每30秒更新）：

| 等级 | 描述中文 |
|------|---------|
| 0 | 不存在 |
| 1 | 陌生人瑟瑟发抖，呢喃不已，听不清在说些什么. |
| 2 | 角落里的陌生人不再颤抖了，她的呼吸平静了下来. |
| 3 | 建造者打量了小屋，微微点了点头. |
| 4 | 建造者说她能帮忙造些东西. |

- 等级4时解锁建造系统（build按钮出现）
- 建造者状态升级需要房间温度 ≥ Warm（3）

### 2.4 森林系统（Outside模块）

**解锁条件**：玩家拥有木头（stores.wood存在）

**操作按钮**：

| 操作 | 冷却 | 效果 |
|------|------|------|
| 收集木头 | 60秒 | 获得10木头（有手推车后50） |
| 检查陷阱 | 90秒 | 根据陷阱数量随机掉落 |

**陷阱掉落表**（原版 `Outside.TrapDrops`）：

每个陷阱独立roll一次（有诱饵的陷阱算两次）：

| 概率 | 掉落 | 文案 |
|------|------|------|
| 0-50% | 毛皮 | 几块毛皮 |
| 50-75% | 肉 | 一些碎肉 |
| 75-85% | 鳞片 | 奇怪的鳞片 |
| 85-93% | 牙齿 | 散落的牙齿 |
| 93-99.5% | 布料 | 破烂的布料 |
| 99.5-100% | 符咒 | 一个粗糙的符咒 |

### 2.5 村庄与人口系统

**人口机制**：
- 每个棚屋容纳 **4人**（原版 `_HUT_ROOM: 4`）
- 有棚屋后，每隔 0.5-3 分钟随机增加人口
- 人口文案（原版）：
  - 1人: "一个陌生人在夜里来了"
  - 2-4人: "一家风尘仆仆的人住进了一间棚屋."
  - 5-9人: "一小群人到来了，满身尘土和疲惫."
  - 10-29人: "一支大篷车队缓缓驶来，满载着忧虑和希望."
  - 30+人: "小镇热闹起来了.消息传得很快."

**工人分配**：
- 人口 = 伐木者（gatherer，默认） + 各种专业工人
- 玩家可以增减各职业的工人数量

**收入系统**（原版 `Outside._INCOME`）：

| trapper | 陷阱师 | 10秒 | -1 肉, +1 诱饵 |
| tanner | 皮革师 | 10秒 | -5 毛皮, +1 皮革 |
| charcutier | 熏肉师 | 10秒 | -5 肉, -5 木头, +1 熏肉 |
| iron miner | 铁矿工人 | 10秒 | -1 熏肉, +1 铁 |
| coal miner | 煤矿工人 | 10秒 | -1 熏肉, +1 煤 |
| sulphur miner | 硫磺矿工 | 10秒 | -1 熏肉, +1 硫磺 |

| armourer | 军械工人 | 10秒 | -1 钢, -1 硫磺, +1 子弹 |

| 建筑 | 解锁工人 |

**贸易商品**（原版 `Room.TradeGoods`）：

| 商品 | 中文 | 成本 | 上限 |
|------|------|------|------|

### 2.7 随机事件系统

**房间事件**（原版 `Events.Room`）——当玩家在"生火间"时可能触发：

1. **游牧商人（The Nomad）** — 条件：有毛皮。可以购买鳞片/牙齿/诱饵/罗盘。
2. **外面有声音（Noises）** — 条件：有木头。随机获得木头或毛皮。
3. **屋内有声音（Noises Inside）** — 条件：有建造者。检查屋内状况。
4. **乞丐（The Beggar）** — 条件：有毛皮。给毛皮获得鳞片。
5. **神秘流浪者（Mysterious Wanderer）** — 条件：有建造者。获得随机物品。
6. **阴影建造者（The Shady Builder）** — 条件：有棚屋。可能失去棚屋。
7. **小偷（Thief）** — 条件：有资源。偷走部分资源。
8. **侦察兵（Scout）** — 条件：有军械库。报告远方消息。
9. **疫病（Plague）** — 条件：有人口。失去部分人口。
10. **野兽袭击（Beast Attack）** — 条件：有人口。战斗或失去人口/棚屋。
11. **士兵袭击（Soldier Attack）** — 条件：有军械库。战斗事件。
12. **病人（Sick Man）** — 条件：有药品。消耗药品救人或获得奖励。
13. **流浪大师（Wandering Master）** — 条件：无。学习技能（perks）。
14. **棚屋起火（Hut Fire）** — 条件：有棚屋。失去棚屋和人口。
15. **被毁的陷阱（Ruined Trap）** — 条件：有陷阱。失去陷阱。

**事件触发机制**：
- 全局事件间隔：普通事件每隔一定时间随机触发
- 原版使用 `Events._EVENT_TIME_RANGE: [3, 6]`（分钟）
- 每次触发时从可用事件池中随机选一个
- 事件以弹窗/模态框形式展示，玩家选择选项

### 2.8 村庄标题（原版 `Outside.setTitle`）

根据棚屋数量动态变化：

| 棚屋数 | 标题 |
|--------|------|
| 0 | 静谧森林 |
| 1 | 孤独的小屋 |
| 2-4 | 小村庄 |
| 5-8 | 中等村庄 |
| 9-14 | 大村庄 |
| 15+ | 喧闹的村庄 |

## 5. 手表适配注意事项

### 5.1 工人分配的交互
原版用 +/- 按钮来分配工人。在手表小屏幕上这个交互很密集。
**建议方案**：每个工人行显示 `[−] 猎人: 2 [+]`，按钮要足够大（至少40x40px），用单行横向布局。

### 5.2 事件弹窗
原版事件是弹窗式的。在手表上做成**全屏覆盖**的事件面板，显示事件文本和选项按钮，选择后关闭回到主界面。

### 5.3 建造按钮的呈现
原版有 build/craft/buy 三个区块。手表上空间有限，可以合并为一个按钮列表，用 `[建造]` `[制造]` `[购买]` 小标签区分分类。或者在按钮上直接标注类型。

### 5.4 触摸适配
- 所有可点击元素最小 44x44 像素
- 按钮间距至少 8px
- 支持圆形屏幕：内容留足边距（至少 10px padding）
- 不要有 hover 效果（手表没有 hover）

## 6. 实现优先级

### P0 — 最小可玩版本（先做这个）

1. 单页UI框架（标签栏 + 按钮区 + 数值区 + 日志区）
2. 火焰系统（添柴、冷却、衰减）
3. 房间温度系统
4. 建造者NPC触发和状态推进
5. 基础资源：木头
6. 建造系统：陷阱、手推车
7. 森林标签页：收集木头、检查陷阱（含掉落表）
8. localStorage 自动存档/读档
9. 日志系统

### P1 — 核心循环完善

10. 更多建筑：棚屋、猎人小屋、贸易站、制革坊、烟熏房、工坊
11. 人口系统（自动增长、文案）
12. 工人分配系统（伐木者/猎人/陷阱师/皮革师/熏肉师）
13. 收入系统（工人自动产出）
14. 制造系统（火把、骨枪、水袋等装备）
15. 随机事件系统（至少实现：游牧商人、外面有声音、乞丐、野兽袭击、小偷）

### P2 — 深度内容

17. 全部建筑和制造配方
18. 全部随机事件
19. 贸易系统
20. 村庄标题动态变化
21. 技能系统（perks）
22. 炼钢/军械产业链
